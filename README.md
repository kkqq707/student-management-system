# 学生管理系统（V1 完整版）

一个基于 `Node.js + Express + 原生前端` 的教务后台项目，覆盖学生、班级、课程、成绩、考勤、系统日志等完整模块。  
当前版本以 **V1 完整可用** 为目标，强调“可演示 + 可测试 + 可扩展”。

## 技术栈

- 后端：`Node.js`、`Express`
- 前端：`HTML`、`CSS`、`Vanilla JavaScript`（函数式组件组织）
- 数据存储：本地 `JSON` 文件持久化
- 会话与安全：`express-session`、`PBKDF2` 密码哈希
- 文件处理：`multer`、`xlsx`

## 核心功能

### 1) 账户与系统
- 管理员登录 / 退出
- 登录失败次数限制与临时锁定
- 修改管理员账号密码（校验旧密码）
- 一键重置默认管理员
- 数据备份与恢复

### 2) 学生管理
- 新增、编辑、删除、批量删除
- 学生详情弹窗
- 学号唯一校验（后端冲突返回 + 前端基础校验）
- 关键词 / 状态筛选、排序、分页
- 表单闭环（编辑回填、取消编辑、提交 loading、保存后刷新与重置）
- Excel 导入、导出、失败行导出

### 3) 班级管理
- 班级新增、编辑、删除
- 班级容量、人数、利用率展示
- 关键词筛选、排序、分页
- 统计卡片（班级数、班主任数、总容量、总人数）

### 4) 课程管理
- 课程新增、编辑、删除
- 关键词筛选、排序、分页
- 统计卡片（课程总数、教师数、总学分、平均学分）

### 5) 成绩管理
- 成绩录入、编辑、删除
- 总评分自动计算（平时 40% + 期末 60%）
- 及格状态自动判定
- 学生 + 课程维度去重约束
- 课程字段联动课程列表（优先下拉选择，兼容历史 `courseName`）
- 支持课程筛选、状态筛选、排序、分页

### 6) 考勤管理
- 考勤新增、编辑、删除
- 按关键词 / 班级 / 状态 / 日期筛选
- 排序、分页、统计（出勤率、缺勤/请假/迟到）

### 7) 看板与日志
- 首页看板：核心卡片 + 班级人数分布 + 成绩状态分布 + 考勤状态分布 + 最近动态
- 系统日志页：日志列表、模块筛选、操作类型筛选、关键词检索、排序、分页、统计卡片
- 兼容旧日志结构（`time/user/detail`）与新日志结构展示

## 页面与交互状态

项目统一支持以下体验状态：
- 表格加载 `skeleton`
- 空数据提示
- 搜索无结果提示
- 接口异常提示
- 操作成功/失败消息
- 删除确认弹窗（单条与批量）

## 本地数据文件

- `data/students.json`：学生、班级、课程、成绩、考勤等业务主数据
- `data/admin.json`：管理员账号与密码哈希
- `data/operation-logs.json`：系统操作日志

## 演示数据说明

项目启动时会在数据不足时自动补齐一套演示数据（不改接口结构），用于验证分页、筛选、统计与看板展示：

- 学生：30+  
- 班级：6+  
- 课程：8+  
- 成绩：80+  
- 考勤：60+  
- 日志：30+  

并保证学生、班级、课程、成绩、考勤、日志之间字段关联一致。

## 启动方式

1. 安装依赖

```bash
npm install
```

2. 启动服务

```bash
npm start
```

3. 浏览器访问

`http://localhost:3000`

## 默认管理员

- 账号：`admin`
- 密码：`admin123`

> 管理员信息首次运行后保存在 `data/admin.json`，密码以哈希方式存储。

## API 说明（节选）

- 学生：`/api/students`、`/api/students/stats`
- 班级：`/api/classes`、`/api/classes/stats`
- 课程：`/api/courses`、`/api/courses/stats`
- 成绩：`/api/grades`、`/api/grades/stats`
- 考勤：`/api/attendance`、`/api/attendance/stats`
- 看板：`/api/dashboard/stats`
- 日志：`/api/logs`、`/api/logs/stats`（兼容旧 `/api/operation-logs`）

## V1 验收要点

- 学生管理闭环：新增/编辑/取消编辑/删除/批量删除/详情
- 多模块 CRUD：班级、课程、成绩、考勤
- 联动能力：课程与成绩联动
- 系统能力：看板、日志、备份恢复
- 统一状态：加载态、空态、错误态、消息反馈
